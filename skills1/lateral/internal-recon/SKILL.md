# 内网信息收集方法论

内网侦察和外网侦察的区别：内网通常没有 CDN/WAF 保护，但你的操作空间受限于当前立足点的网络位置。核心目标是**快速绘制内网地图**——有哪些主机、什么服务、哪些是高价值目标。

## Phase 1: 当前位置分析

在开始扫描之前，先搞清楚自己在哪里：

### 1.1 网络信息
```bash
# Linux
ip addr                     # 网络接口（可能有多块网卡→多个网段）
ip route                    # 路由表（能到达哪些网段）
cat /etc/resolv.conf        # DNS 服务器（域环境中通常指向域控）
arp -a                      # ARP 缓存（已经通信过的主机——免费情报）
cat /etc/hosts              # 静态解析（可能有内部主机名映射）

# Windows
ipconfig /all               # 网络配置 + DNS + DHCP
route print                 # 路由表
arp -a                      # ARP 缓存
type C:\Windows\System32\drivers\etc\hosts
net view /domain            # 列出域
```

### 1.2 关键判断
- **多网卡** → 当前主机跨多个网段，是天然跳板
- **DNS 指向内网 IP** → 大概率是域环境
- **ARP 表有大量条目** → 活跃网段，很多主机
- **路由表有多条路由** → 能到达多个子网

## Phase 2: 存活主机发现

### 2.1 快速存活探测
```bash
# ICMP ping 扫描（最快，但可能被防火墙阻止）
nmap -sn 10.0.0.0/24

# ARP 扫描（同网段最可靠，不受防火墙影响）
nmap -PR -sn 10.0.0.0/24

# 用 fscan（国产，快速全面）
fscan -h 10.0.0.0/24 -nopoc
```

### 2.2 多网段场景
从路由表和网络配置推断可达网段，逐段扫描：
```bash
# 常见内网网段
10.0.0.0/24, 10.0.1.0/24, 10.0.2.0/24
172.16.0.0/24, 172.16.1.0/24
192.168.1.0/24, 192.168.2.0/24
```

## Phase 3: 端口扫描与服务识别

对存活主机做端口扫描，**聚焦关键端口**（全端口扫描太慢）：

```bash
nmap -sT -p 22,80,135,139,443,445,1433,3306,3389,5432,5985,6379,8080,8443,9200 TARGETS
```

### 端口→服务→价值映射
| 端口 | 服务 | 攻击价值 |
|------|------|----------|
| 88 | Kerberos | **域控**（最高价值） |
| 389/636 | LDAP/LDAPS | **域控** |
| 445 | SMB | 文件共享、横向移动入口 |
| 135 | RPC | Windows 主机、WMI |
| 3389 | RDP | 远程桌面 |
| 5985/5986 | WinRM | 远程管理 |
| 1433 | MSSQL | 数据库（可能有 xp_cmdshell） |
| 3306 | MySQL | 数据库 |
| 5432 | PostgreSQL | 数据库 |
| 6379 | Redis | 无认证→RCE |
| 9200 | Elasticsearch | 无认证→数据泄露 |
| 22 | SSH | Linux 远程访问 |
| 80/443/8080 | HTTP | Web 应用 |
| 27017 | MongoDB | 无认证→数据泄露 |

## Phase 4: 关键基础设施定位

### 4.1 域控识别
- 开放 88+389+445 = 几乎确定是域控
- DNS 服务器 IP 通常就是域控
- `nslookup -type=SRV _ldap._tcp.dc._msdcs.DOMAIN`

### 4.2 数据库服务器
- 1433(MSSQL) / 3306(MySQL) / 5432(PostgreSQL) / 1521(Oracle)
- 用收集到的凭据尝试连接

### 4.3 管理系统
- 指纹识别 Web 服务（`scan_finger`）→ 发现 Jenkins/GitLab/Zabbix 等
- 这些系统通常有弱密码或已知漏洞

## Phase 5: 输出内网地图

整理发现为结构化报告：
1. **网络拓扑**：网段划分、主机分布、跨网段路径
2. **高价值目标清单**：域控→数据库→管理系统→跳板机
3. **已发现的攻击面**：弱密码服务、未认证数据库、Web 应用
4. **下一步建议**：优先攻击路径

## 注意事项
- 内网扫描会产生大量流量，如果有 IDS/SIEM 可能会被发现
- 优先用 ARP 缓存和路由表做**被动发现**，再做主动扫描
- 分批扫描，避免一次扫描整个 /16 导致网络问题
- 将发现的资产用 `save_credential`/`memory_save` 保存
