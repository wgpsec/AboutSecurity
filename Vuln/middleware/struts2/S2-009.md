## 漏洞描述

Struts2 对 S2-003 的修复方法是禁止静态方法调用，在 S2-005 中可直接通过 OGNL 绕过该限制，对于 # 号，同样使用编码 \\u0023 或 \\43 进行绕过；于是 Struts2 对 S2-005 的修复方法是禁止 \\ 等特殊符号。但是，如果当前 action 中接受了某个参数，这个参数将进入 OGNL 的上下文，可以通过特殊方式绕过限制。

## 影响版本

- Apache Struts 2.1.0 - 2.3.1.1

## 前置条件

- 无需认证
- 目标应用存在接受字符串参数的 action

## 利用步骤

1. 找到接受参数的 action
2. 将 OGNL 表达式放在参数中
3. 使用特殊绕过方法执行

## Payload

```http

Host: target:8080
```

## 修复建议

1. 升级 Apache Struts 至 2.3.2+
2. 对用户输入进行严格过滤
3. 避免在业务逻辑中直接使用用户参数进行 OGNL 解析
